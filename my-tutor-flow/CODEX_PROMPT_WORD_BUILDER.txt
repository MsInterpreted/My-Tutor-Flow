SECTION 2 — FEATURE OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The Word Builder is a 100-level progressive vocabulary programme.
Students climb through four stages by completing levels in sequence.
Each level introduces a set of new words and puts them through five
interactive activities. Progress is saved to Firebase Firestore
(when authenticated) and localStorage (always, as fallback/cache).
Teachers can watch their students' progress update in real time from
the Teacher Dashboard without refreshing the page.

There are NO age labels anywhere in the UI. Levels communicate
progress entirely. A student's current level tells the story.

SECTION 3 — LEVEL ARCHITECTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Four stages, 100 levels total. Stages unlock sequentially.

┌─────────────────┬──────────────┬────────────────┬─────────────────┐
│ Stage           │ Levels       │ Words/level    │ Review pattern  │
├─────────────────┼──────────────┼────────────────┼─────────────────┤
│ Beginner        │  1 – 10      │ 6 new words    │ Level 5, 10     │
│ Intermediate    │ 11 – 30      │ 8 new words    │ Every 5th       │
│ Pre-Advanced    │ 31 – 60      │ 10 new words   │ Every 5th       │
│ Advanced        │ 61 – 100     │ 12 new words   │ Every 5th       │
└─────────────────┴──────────────┴────────────────┴─────────────────┘

REVIEW LEVELS
Every 5th level within a stage is a REVIEW LEVEL — it contains no
new words. Instead it replays all words from the previous 4 levels
in harder activity combinations. Review levels are marked with a
special recycling icon on the game map.
  Example: Level 5 reviews words from Levels 1–4.
           Level 10 reviews all of Levels 1–9 (stage finale).
           Level 15 reviews words from Levels 11–14.

WORD COUNTS
New-word levels only (review levels use previous words):
  Beginner:     8 new-word levels × 6  =   48 unique words
  Intermediate: 16 new-word levels × 8  =  128 unique words
  Pre-Advanced: 24 new-word levels × 10 =  240 unique words
  Advanced:     32 new-word levels × 12 =  384 unique words
  ─────────────────────────────────────────────────────────
  TOTAL unique words in bank:             800

STAR RATINGS PER LEVEL
After completing all activities in a level, award stars:
  < 60%  = 0 stars  (must replay before unlocking next level)
  60-79% = 1 star   ⭐
  80-89% = 2 stars  ⭐⭐
  90-99% = 3 stars  ⭐⭐⭐
  100% first attempt = GOLD STAR  🌟  (special badge trigger)

STAGE TRANSITIONS
Completing the final level of a stage triggers a full-screen
STAGE COMPLETE celebration (animated trophy, confetti, stat summary)
before the next stage map is revealed.

ADAPTIVE FAST-TRACK
If a student earns 3 stars on 3 consecutive levels without replaying,
a "You're flying! ⚡ Skip ahead 2 levels?" prompt appears.
Skipping is optional and parents can disable it in settings.

SECTION 6 — PLACEMENT TEST
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Route: /words/placement

Shown to a user the FIRST time they open the Word Builder (tracked
in localStorage / Firestore). Can be re-taken from settings.

Structure:
  - 15 adaptive MCQ questions
  - Questions start at Beginner difficulty
  - If student answers 4+ correctly in a row, questions step up a stage
  - If student answers 2 wrong in a row, questions step down
  - After 15 questions the algorithm places the student at a starting level:
    ≤ 5 correct → Level 1 (Beginner)
    6–9 correct → Level 5 (mid-Beginner)
    10–12 correct → Level 11 (Intermediate start)
    13–14 correct → Level 20 (mid-Intermediate)
    15 correct all at Pre-Advanced → Level 31 (Pre-Advanced start)

Question format is identical to the Grammar Check activity (single sentence,
4 options). Questions must cover a mix of: definition match, synonym choice,
word-in-context, fill-in-the-blank sentence.

After the test, show:
  "We've placed you at Level [N] — [Stage Name]"
  A brief explanation of why: "You got the Intermediate questions right
  but found the Pre-Advanced ones tricky. Level 20 is a great starting point."
  Large "Start Level [N]" button.

The placement result is stored as startingLevel in the user's progress.

VISUAL DESIGN
The level select screen is a VERTICAL SCROLLING GAME MAP — a winding
path with circular nodes for each level, like a trail through four
distinct landscape zones:

  Beginner zone (Levels 1–10):
    Visual theme: green meadow, bright sunshine, simple flora
    Colour palette: teal (#0D9488) tones

  Intermediate zone (Levels 11–30):
    Visual theme: a town with buildings and bridges
    Colour palette: purple (#7C3AED) tones

  Pre-Advanced zone (Levels 31–60):
    Visual theme: mountainous terrain, more complex and dramatic
    Colour palette: blue (#2563EB) tones

  Advanced zone (Levels 61–100):
    Visual theme: a city at night with glowing windows and stars
    Colour palette: gold (#D97706) tones

Each level node shows:
  - Level number
  - Stars earned (0–3, or gold star 🌟)
  - Lock icon (grey) if not yet reachable
  - Review level nodes use a ♻️ recycling symbol instead of a number
  - Current level pulses with a soft glow animation
  - Tapping a completed node replays it; tapping locked nodes shows
    "Complete Level [N-1] to unlock"

At the TOP of the screen (sticky header):
  - Student name + avatar
  - Current level badge: "Level 23 · Intermediate"
  - Overall XP bar
  - Streak counter with flame: "🔥 7 days"
  - "Continue →" button jumps straight to current level

Zones are separated by animated BRIDGE segments — a visual milestone
the student sees when moving between stages.

A level session has FIVE STAGES shown sequentially. A thin progress
bar at the top of the screen shows Stage 1–5. Each stage has a
distinct visual identity and animation style.

─────────────────────────────────────────
STAGE 1 — WORD REVEAL
─────────────────────────────────────────
One word at a time. For each word in the level:

CARD FRONT (initial view):
  ┌───────────────────────────────────────┐
  │  [emoji]   curious          🔊        │  ← tap speaker for TTS
  │  adjective · cu·ri·ous                │
  │  KYOOR-ee-us                          │
  │  ─────────────────────────────────    │
  │  eager to know or learn               │
  │  about something                      │
  │                                       │
  │  [example 1 in light tinted box]      │
  │  [example 2 in light tinted box]      │
  │  ─────────────────────────────────    │
  │  Also: curiosity · curiously          │  ← word family chips
  │  ─────────────────────────────────    │
  │  Synonyms: inquisitive · interested   │  ← pill badges

TTS: use window.speechSynthesis with lang 'en-GB' if available.
     Trigger on card appear AND on speaker tap.

After all words are revealed, auto-advance to Stage 2 with a
"Now let's practise!" transition screen (1.5 seconds, animated).

─────────────────────────────────────────
STAGE 2 — SENTENCE TASK
─────────────────────────────────────────
One word at a time. Varies by level:

BEGINNER (Levels 1–10) — STRUCTURED SENTENCE BUILDER:
  Do NOT show a free-text area. Instead show a word-bank sentence
  completion activity:
  - Display a sentence with one blank: "The ___ cat hid under the bed."
  - Show 4 word-bank buttons below (one is the correct word, three
    are plausible distractors from the same level)
  - On correct tap: green flash, sentence completes, audio cheer
  - On wrong tap: gentle red shake, hint appears: "Think about the
    definition: [simpleDefinition]"

INTERMEDIATE AND ABOVE — FREE WRITE:
  - Large text area: "Write your own sentence using [word]"
  - Character minimum: Intermediate 15 chars, Pre-Advanced 25 chars,
    Advanced 40 chars (with a character counter shown)
  - A "Hint" button reveals a sentence starter scaffold, e.g.:
    "Try: 'She was [word] because...'"  or  "Try: 'The most [word]
    thing about the situation was...'"
  - After submitting: "Nice sentence! ✓" with a sparkle animation
  - Sentences saved to localStorage AND Firestore (see Section 10)
  - Do NOT grammar-check free writing here — this is creative output

─────────────────────────────────────────
STAGE 3 — GRAMMAR CHECK
─────────────────────────────────────────
8 sentences drawn from all the current level's words (grammarCheck
arrays). Mix correct and incorrect sentences roughly 50/50.

Each sentence card shows:
  - The sentence in large readable type
  - Two big tap targets: ✅ CORRECT   ❌ INCORRECT
  - Countdown timer bar (10 seconds — purely visual, no penalty for
    taking longer, but completing under 5s awards a "Speed" bonus point)

After tapping:
  - Correct answer: green card flash + "That's right! ✓"
    If the sentence WAS incorrect, show corrected version:
    • Wrong part highlighted in red strikethrough
    • Corrected part highlighted in green
    • One-line explanation of the grammar error type
  - Wrong answer: red shake + correct answer revealed immediately +
    explanation shown

Streak counter: "🔥 4 in a row!" fires a confetti burst at streaks
of 3, 5, and 8.

End of stage: score card "7 / 8 — Grammar Star! ⭐" with animated
badge if 8/8.

─────────────────────────────────────────
STAGE 4 — CONTEXT CLUES  (Intermediate and above only)
         MATCHING GAME  (Beginner — no context clues)
─────────────────────────────────────────
FOR BEGINNER — MATCHING GAME ONLY (simple):
  Two columns, 6 pairs. Tap a word on the left, tap its definition
  on the right. Correct pairs lock green. Wrong pairs shake red.
  No timer. Encouraging only.

FOR INTERMEDIATE AND ABOVE — TWO-PART STAGE:

  PART A — CONTEXT CLUES  (one per level session):
  Show a short passage (3–5 sentences) with one word blanked [BLANK].
  Four options below. Student taps the word that fits.
  Use the contextClue field from the word data.
  Show the explanation after answering, whether right or wrong.

  PART B — MATCHING GAME  (all levels):
  Word-to-definition drag or tap-to-match.
  Left column: words. Right column: definitions (shuffled).

  BONUS ROUND (Pre-Advanced and Advanced):
  After standard matching completes, a second round appears:
  "Now match each word to its SYNONYM"
  Uses synonyms array. Harder — no definition hint this time.

  BONUS ROUND 2 (Advanced only):
  "Match the word to its ANTONYM"

Scoring: +15 per correct match, −5 per incorrect attempt.
         Part A context clue: +20 if correct first try, +10 second try.

─────────────────────────────────────────
STAGE 5 — SESSION COMPLETE
─────────────────────────────────────────
Full-screen celebration card:

  [Animated trophy or star burst at top]

  Level [N] Complete!
  ⭐⭐⭐  (earned stars, animated fill-in)

  This session:
    [score]% accuracy   [XP earned] XP   [time taken]

  Your streak: 🔥 [N] days in a row

  [XP bar filling toward next badge, with badge name shown]

  [Grid of today's word pills — tap any to flip open its Word Card]

  [Buttons:]
    "Save to Vocabulary Journal"  →  stores session to journal
    "Share"  →  copies text to clipboard:
      "I mastered [N] new words on MyTutorFlow today! 💪
       curious · resilient · elaborate · [etc]"
    "Back to Map"  →  returns to /words game map, level node
                       now shows earned stars

SPELLING CHALLENGE  (appears after every 5th completed level)
  Before going back to map, an optional bonus:
  "Want a quick spelling challenge?"  [Yes]  [Skip]

  Each word shown for 3 seconds → disappears → student types it.
  Correct = green sparkle. Wrong = shows word with each mismatched
  letter highlighted in orange. No score impact — pure practice.
  Completing all words = "Spelling Champion!" badge trigger check.

REVIEW LEVELS — DIFFERENT FLOW
For review levels (every 5th), all 5 stages run but source words
from ALL previous levels in the stage. The Word Reveal stage is
replaced with a "How well do you remember?" challenge:
  - Show the emoji + word, student taps the correct definition from 4 options
  - No TTS re-read (they should know it by now)
  - Harder matching (more words, smaller cards)
  - No new context clues — use a mix of all previous ones
  - Completing a review level with 3 stars unlocks a special
    "Consistent Learner" badge.

REAL-TIME TEACHER READS
Teachers read student data using onSnapshot (real-time listener):
  /users/{studentUid}/wordBuilderStats   — live stats
  /users/{studentUid}/wordBuilderSentences — all saved sentences

The teacher dashboard listens to ALL students in their class.
Class membership is stored at:
  /classes/{classId}/studentIds: string[]    (array of UIDs)
  /classes/{classId}/teacherId: string       (teacher UID)

On mount, the teacher component:
  1. Reads /classes/{classId}/studentIds
  2. Fetches each student's display name from /users/{uid}/displayName
  3. Attaches onSnapshot listener to each student's wordBuilderStats
  4. Cleanup all listeners on unmount

This gives the teacher a live-updating table without any refresh.

SECTION 17 — ANIMATION & DESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

General rules:
  - Tailwind transition/animation utilities throughout
  - No heavy animation libraries — use CSS keyframes via Tailwind's
    @keyframes in index.css if custom animations are needed
  - For confetti on badge awards and Stage 5: use canvas-confetti
    (add to package.json: "canvas-confetti": "^1.9.3")
  - All interactive elements must have :hover, :focus-visible, and
    :active states
  - Fully responsive: works from 360px mobile to 1440px desktop
  - Dark mode: read theme from useTheme() context, NOT from a class check

Specific animations required:
  - Word card entrance: fade-in + slide up (150ms ease-out)
  - Correct answer: green pulse ring expanding outward (300ms)
  - Wrong answer: horizontal shake keyframes (300ms)
  - Grammar Check streak fire: bouncing flame emoji (loop while active)
  - Level complete stars: each star flips in sequentially 200ms apart
  - XP bar fill: smooth width transition (800ms ease-in-out)
  - Badge modal: scale from 0.8 + fade in (250ms) + confetti burst
  - Stage transition: slide left (300ms ease-in-out)
  - Level map: locked nodes are desaturated; unlocked nodes pulse
    gently (subtle 2s loop scale 1.0→1.04→1.0)
  - Bridge/zone transition: brief sparkle sweep animation
  - Live teacher table row update: ring flash (yellow → transparent, 1s)

Typography:
  - Word titles on cards: text-4xl font-bold (or larger on wide screens)
  - Definitions: text-lg for Beginner, text-base for others
  - Syllable display: monospace font, letter-spacing-wide
  - Example sentences: italic, slightly lighter text colour

SECTION 18 — TECHNICAL REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  - React hooks only (no class components)
  - No new npm packages except canvas-confetti
  - Touch events for drag interactions (no drag-and-drop library).
    Matching game defaults to tap-to-select (tap word, tap definition).
    Drag is progressive enhancement via pointer events if time allows.
  - Web Speech API for TTS — always check 'speechSynthesis' in window
    before calling. Graceful no-op if unavailable (hide speaker icon).
  - Firestore writes use try/catch — never crash if Firestore is
    unavailable. Fall through to localStorage silently.
  - All localStorage reads use try/catch (corrupted data must not crash).
  - The word bank file will be large (~800 entries). Export it as a
    default export array for tree-shaking. Do not import the entire bank
    into components — filter by level number at the hook level and
    pass only the needed words down.
  - Daily determinism: to select words for a session use
    the level number as the source — levels are fixed, not rotated daily.
    (Daily rotation is replaced by level progression.)
